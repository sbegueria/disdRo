disdRo: a package for reading and handling raw disdrometer data

Santiago Beguería

24 May 2016

Introduction

disdRo is an R package for reading and basic handling of raw disdrometer data.

One day of raw data from a Thies Klima LPM disdrometer are provided in directory tree under extdata/rawDataThies, in the package directory. You can find the location of those files with system.file('extdata', 'rawDataThies', package='disdRo'). Similarly, sample data from a OTT Parsivel2 disdrometer can be found in directory extdata/rawDataParsivel.

The didRo package contains some R functions for reading and handling rainfall drop size and velocity distribution (DSVD), or disdrometric matrix data, from the raw data files generated by Thies LPM or Parsivel optical disdrometers.

A bunch of Perl scripts are also included under perl for computing integrated variables from the raw data files. These scripts would eventually be ported into R.

Example data files are stored on extdata.

Working with the DSVD matrix

We shall load the library to start:

library(disdRo)
## Loading required package: ggplot2
## Loading required package: reshape2
## Loading required package: abind
## Loading required package: akima

The function read_dsd() retrieves minutal DSVD data from a list of raw disdrometer data files. For each minute, a matrix of velocity (rows) vs. size (columns) drop counts is provided.

Example with Thies LPM data

f <- system.file('extdata', 'rawDataThies', package='disdRo')
files <- list.files(f, '.txt', full.names=TRUE, recursive=TRUE)
DSVD <- dsd_read(files, type='Thies')
dim(DSVD)
## [1] 1439   20   22

The function dsd_plot() produces a drop count velocity vs. size plot from the DSVD data. The function interprets DSVD matrices from either Thies LPM or OTT Parsivel devices, being Thies the default option.

dsd_plot(day, type='Thies')

It is possible to choose the theoretical models to plot on top of the DSVD data:

dsd_plot(day, model='VanDijk')

Or to plot no models at all:

dsd_plot(day, model=NA)

The size of the bins in the previous plot correspond to those of the raw matrix yielded by the disdrometer. Interpolated 2D desnity contour lines can be plotted on top by setting parameter contour to TRUE:

dsd_plot(day, model=NA, contour=TRUE)

Example with Parsivel data

Data can also be read from a Parsivel or Parsivel2 disdrometer. In this case, the parameter type of dsd_read() and dsd_plot() functions can not be overriden, since Parsivel type is not the default option.

f <- system.file('extdata', 'rawDataParsivel', package='disdRo')
files <- list.files(f, '.txt.gz', full.names=TRUE, recursive=TRUE)
DSVD <- dsd_read(files, type='Parsivel')

day <- apply(DSVD, c(2,3), sum)
barplot(colSums(day), main='Daily drop counts per size class')

barplot(rowSums(day), main='Daily drop counts per velocity class')

time <- apply(DSVD, 1, sum)
barplot(time, main='Number of drops per minute')

dsd_plot(day, type='Parsivel')

Working with the integrated variables

process-*.pl son una serie de scripts en Perl para leer datos de disdrómetro y extraer las variables integradas, así como calcularlas a partir de la matriz de DSVD. Cada script está optimizado para uno de los disdrómetros:

  • process-mastil-p1 datos de un disdrómetro Parsivel instalado en la EEAD
  • process-formigal-p1.pl para datos de los disdrómetros Parsivel instalados en Formigal
  • process-p1.pl para datos del disdrómetro Thies 0436
  • process-p2.pl para datos del disdrómetro Thies 0655 (con anemo y termo)
  • process-p3.pl para datos del disdrómetro Parsivel 304555
  • process-p4.pl para datos del disdrómetro Parsivel 304553
  • process-anillo.pl datos de ocho disdrómetros Parsivel2 colocados en anillo

Resulta sencillo obtener gráficas de las distintas variables, una vez formateados los tiempos como POSIXct.

int$time <- as.POSIXct(int$time)
par(mfrow=c(3,1))
plot(int$r~int$time, type='l', xlab='', ylab='I (mm/h)',
     main='Precipitation rate')
plot(int$e~int$time, type='l', xlab='', ylab='ET (J m-2 mm-1)',
     main='Unit kinetic energy')
plot(int$z~int$time, type='l', xlab='', ylab='Z (dB mm6 m-3)',
     main='Radar reflectivity')

Convirtiendo el objeto leído (un data.frame) en un objeto tipo zoo (serie temporal), resulta muy sencillo realizar operaciones con respecto al tiempo, por ejemplo extraer un evento determinado.

library(zoo)
int <- zoo(int[,-c(1:3)], as.POSIXct(int$time))

event <- window(int, start='2016-05-18 09:30:00', end='2016-05-18 11:30:00')
plot(event[,c('r_meas','z_meas','d50')], type='l', xlab='', main='A precipitation event')
